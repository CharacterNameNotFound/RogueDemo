# RogueDemo

Demo for autobattler project, currently have only:

Basic systems, such as:

* Ui management (with auto-injection of dependencies) and prebuild generic state-based class for complex UIs
* Universal input layer, 
* Game state manager, 
* Localization manager, Etc



Basic gameplay circle implemented. In-game item creation through building game objects, with conversion to jsons, for easy item creation without writing a per-item script, and future compatibility if the redactor UI is introduced
